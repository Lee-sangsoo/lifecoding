1. 웹 페이지 열기 -> Ctrl + O

2. 문자 강조 -> <strong> ... </strong> : 태그

3. 밑줄 치기 -> <u> ... </u> : 태그

4. 태그 종류 확인하기 -> w3schools.com/tags
	https://www.advancedwebranking.com/html/ -> 이 사이트를 통해 자주 세계적으로 자주 쓰이는 태그 20개 정도 확인 가능하다.

5. <h1> to <h6>  -> 숫자가 커질수록 글씨가 작아진다.

6. 줄바꿈 태그 -> <br>
  ㄴ 줄바꿈과 같이 무엇인가 설명하지 않는 태그들은 감싸야하는 컨텐츠가 없기 때문에 태그를 닫지 않아도 된다.

7. 단락 태그 -> <p> </p>  (paragraph)

8. <br>과 <p>를 비교했을 때, <p>의 경우 단락간의 간격이 고정되어 있기 때문에 한계가 있다.
   하지만, <p style="margin-top:45px;"> 와 같이 CSS를 적용 시키면 이 한계를 극복 가능하다.

9. 태그만으로 모든 정보를 표현하지 못하는 경우 -> 속성 사용
      ㄴ 이미지 -> <img src="주소">
      ㄴ 이미지를 프로젝트 폴더에 추가할 경우 -> 이미지의 이름.확장자 만 src에 적어주면 된다.

10. 저작권이 없는 무료 이미지 다운 -> https://unsplash.com/

11. 이미지의 크기 조절 -> <img src="주소" width="100%">

12. 목차를 표현하는 태그 -> <li> </li>

13. 목차를 구분짓는 태그 -> <ul> </ul>       (unorderes list)
     ㄴ <ul>과 <li>는 부모 자식 관계의 태그이다. ==> <ul> <li>   </li> </ul>

14. 목차에 대해 자동 번호 부여를 해주는 태그 -> <ol> <li> </li> </ol>       (ordered list)

15. 웹 페이지의 제목(상단 탭바의 내용) 태그 -> <title> </title>

16. 코딩된 문자방식을 알려주어야 문자가 깨지지 않는다 그러므로 페이지에 알려주자
	ㄴ <meta charset="utf-8">

17.  본문 태그 -> body
      본문을 설명하는 태그 -> head

18. head -> title, meta 등
19. body -> ol, p 등
20. 이 두가지 body와 head를 감싸는 구조 태그 -> html
      ㄴ 단, html 태그는 해당 페이지가 html로 만들어 졌다는 표시를 위해
	<!doctype html>   </html> 과 같이 적어준다.

21. 링크를 설정해주는 태그 <a> ... </a>
       ㄴ <a href="주소" target="_blank" title="html5 specification"> ... </a>
	ㄴ target="_blank" 는 링크 클릭시 새창에서 페이지가 열리게 하는 속성
	ㄴ title의 내용은 해당 링크가 어떤 내용인지에 대한 툴팁을 보여주는 기능

22. 클라이언트와 서버는 고객과 사업자의 비유이다.
	게임을 이용하는 사람의 컴퓨터 프로그램 = 게임 클라이언트
	게임을 운영하는 회사의 컴퓨터 프로그램 = 게임 서버

	채팅을 이용 = 채팅 클라이언트 // 채팅을 운영 = 채팅 서버
	ㄴ 인터넷을 이용하는 모든 정보 시스템에 적용되는 문법이다.

23. github는 웹 호스팅을 제공한다 
	ㄴ 레포지토리 생성 -> 웹 페이지 파일 업로드 -> commit -> setting -> GitHub Pages
	    -> Source -> branch 변경 -> GitHub Pages 상단에 주소가 생성 -> 해당 주소에서 호스팅 진행

24. 처음 레포지토리를 만들 때, README를 만들지 않으면 워킹 디렉토리를 만들지 않아 파일 업로드 버튼이 존재하지 않는다.

25. 기본적으로 웹 서버를 운영할 수 있는 프로그램으로는 아파치가 있다. 
	아파치는 윈도우에 까는게 힘든데 이걸 도와주는 프로그램들이 많다
	"WAMP"로 설치 가능

26. 아파치 웹 서버를 실행 한 다음 "localhost" "localhost/index.html" "127.0.0.1/index.html" 모두 같은 화면이 표시된다.

27. 아파치 서버의 index.html 파일은 C->bitnami->wampstack->apache->htdocs 에 있다.

28. http://자기IP주소/index.html 로 접속 가능 -> 같은 네트워크로 연결되어 있어야 한다는 단점

29. disqus를 사용하여 댓글 기능을 추가 가능하다 (무료) (웹 서버 통해야 사용 가능)

30. tawk.to 를 통해 채팅 기능을 추가 가능하다 (무료) (웹 서버 통해야 사용 가능)

31. <a> 태그에는 id가 존재한다. id속성을 통해 같은 페이지 내에서 책갈피를 사용 가능하다.
	ㄴ 이동할 위치 지정 -> id = "이름"
	ㄴ 누르는 곳 속성 -> href = "#이름"

32. 다양한 태그들이 있다. 이를 위해 추가적으로 검색해 가며 공부할 필요가 있다.
